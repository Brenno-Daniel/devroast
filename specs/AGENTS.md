# Specs

Formato para especificações antes de implementar features.

## Estrutura

Cada spec em arquivo `.md` com:

1. **Objetivo** - O que resolver
2. **Decisões (V1)** - Acordos de produto
3. **Arquitetura** - Como implementar (rascunho)
4. **Riscos** - O que pode dar errado + mitigações
5. **To-dos** - Passos para implementação
6. **Referências** - Links relevantes

## Regras

- Antes de codar, criar spec em `specs/`
- Usar nome descritivo: `code-editor-syntax-highlight.md`
- Manter conciso; detalhes em seções se necessário
- Revisar com o time antes de implementar

Ver `code-editor-syntax-highlight.md` como modelo.
